# Corrupted LZ77 Recovery

## Task
You are given 30 compressed binary files (`corrupted_xxx.bin`).
Each file was generated by compressing a 1600-byte payload using the custom LZ77 algorithm provided in `compressor.py`.
However, after compression, **exactly 3 random bytes** in the compressed stream were mutated (flipped to a random different value).

## Structure of the Original Data
The original 1600-byte uncompressed data consisted of 100 blocks.
Each block is exactly 16 bytes:
- Bytes 0..13: Payload data.
- Bytes 14..15: A Modbus CRC16 of the payload data (see `crc16` in `compressor.py`).

## Goal
For each corrupted file, recover the original 1600-byte uncompressed data.
Output the items in JSONL format, containing exactly:
`{"id": "item_xxx", "answer": "hex_encoded_sha256_of_1600_bytes"}`

## Hints
- Blind brute-force of all $\binom{N}{3} \times 255^3$ byte flips will be too slow.
- The dense CRC16 checks embedded in the original data allow for extremely aggressive pruning during decompression.
- If your decompression outputs 16 bytes and the CRC16 fails, you know a mutation occurred in the compressed tokens that generated this block.
